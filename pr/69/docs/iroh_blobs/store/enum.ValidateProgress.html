    },
    Entry {

        size: u64,
    },
    EntryProgress {

        offset: u64,

    EntryDone {

        error: Option<String>,
    },

    },
    PartialEntryProgress {
        id: u64,
        offset: u64,
    },

    },

    Abort(Error),
}

Progress updates for the validate operation

Variants§

§

Starting

started validating

Fields

§total: u64

The total number of entries to validate

§

Entry

We started validating a complete entry

Fields

§id: u64

a new unique id for this entry

§hash: Hash

the hash of the entry

§path: Option<String>

location of the entry.

In case of a file, this is the path to the file. Otherwise it might be an url or something else to uniquely identify the entry.

§size: u64

The size of the entry, in bytes.

§

EntryProgress

We got progress ingesting item id.

Fields

§id: u64

The unique id of the entry.

§offset: u64

The offset of the progress, in bytes.

§

EntryDone

We are done with id

Fields

§id: u64

The unique id of the entry.

§error: Option<String>

An error if we failed to validate the entry.

§

PartialEntry

Fields

§id: u64

a new unique id for this entry

§hash: Hash

the hash of the entry

§path: Option<String>

location of the entry.

In case of a file, this is the path to the file. Otherwise it might be an url or something else to uniquely identify the entry.

§size: u64

The best known size of the entry, in bytes.

§

PartialEntryProgress
